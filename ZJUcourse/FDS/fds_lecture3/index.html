    TREES¶

    1 Preliminaries¶

    什么是树¶

    树是节点的集合，一般由根节点、零个或多个非空的子树组成。

    说明¶

    一颗树有 \(n\) 个节点，那么它有 \(n-1\) 条边。

    相关定义¶

    • Degree of a node 节点的度，指这个节点拥有的子树的数量

    • Degree of a tree 树的度，指树中所有节点度的最大值

    • Parent node 父节点，拥有子树的节点

    • Children node 子节点，某个父节点下子树的根节点

    • Siblings node 兄弟节点，拥有同一个父节点的子节点

    • Leaf / Terminal node 叶子节点，度为 0 的节点

    • Path from \(n_1\) to \(n_k\) 一个唯一的节点序列

    • Length of path 路径长度，指该路径上边的数量

    • Depth of \(n_i\) 节点深度，指从根节点到该节点的路径的长度

    • Height of \(n_i\) 节点高度，指从该节点到叶子节点的最长路径的长度

    • Height / Depth of a tree 等于根节点的高度，也等于最深的叶子结点的深度

    • Ancestors 从该节点向上到根节点路径上的所有节点

    • Descendants 该节点子树中的所有节点

    实现方法¶

    List Representation¶

    类似嵌套集合。

    这样列表无法确定一个节点有多少子节点，需要多少指针，内存分配不稳定。

    FirstChild-NextSibling Representation¶

    每个节点有两个指针，左指针指向它的第一个子节点，右指针指向它的亲兄弟。当然表示方法不唯一。

    这种类似二叉树的结构，保证了每个节点的大小一致。

    2 Binary Trees¶

    定义¶

    二叉树的任何节点拥有的子节点数量不超过 2 。

    表达式树¶

    把后缀表达式转为表达式树的规则是：

    • 遇到操作数，创建叶子节点并入栈

    • 遇到运算符，从栈中弹出两个节点，先弹出的作为右孩子，后弹出的作为左孩子，创建以当前运算符为根的新树并入栈

    • 从左到右遍历完后缀表达式，最后栈中剩下的一个节点就是表达式树

    遍历二叉树¶

    Preorder Traversal¶

    前序遍历，中左右。

    Postorder Traversal¶

    后序遍历，左右中。

    Inorder Traversal¶

    中序遍历，左中右。

    思路

    以上三种遍历都可以基于递归。

    Levelorder Traversal¶

    层序遍历，从根开始一层一层向下，从左至右遍历。

    一般树的遍历

    对于一般的树不会使用中序遍历，因为较难定义中间的节点。

    Threaded Binary Trees¶

    普通二叉树里，有些节点、尤其是叶子节点的指针是空的，浪费内存。线索二叉树把这些空指针利用起来，其中：

    • Predecessor 前驱线索，即如果左指针为空，就让它指向中序遍历中的前一个节点；

    • Successor 后继线索，即如果右指针为空，就让它指向中序遍历的后一个节点。

    在实际代码实现中，还可以加上标记区分一个指针是指向孩子还是指向线索；以及加上 head node 使所有没有前驱或后继的线索有去向，整体自洽。

    公式

    • 基本公式：

      • 一颗有 \(n\) 个节点的树，边数为 \(n-1\)

      • 所有节点的度数之和等于边数

      • 一颗度为 \(k\) 的树，设 \(n_i\) 为度为 \(i\) 的节点数，则叶子节点数 \(n_0 = 1 + \sum_{i=1}^k (i-1)n_i\)

    • 二叉树适应的公式：

      • 从根节点第 0 层开始，第 \(i\) 层最多有 \(2^i\) 个节点

      • 深度为 \(k\) 的二叉树，虽多有 \(2^{k+1} - 1\) 个节点

      • 叶子节点和度为 2 的节点满足 \(n_0 = n_2 + 1\)

    3 Binary Search Trees¶

    定义¶

    BST 是一颗二叉树，可以为空；如果不为空，那么具有以下性质：

    • 每个节点都有一个整数值，并且没有重复

    • 左孩子的值小于根的值，右孩子的值大于根的值

    • 左子树和右子树仍然是 BST

    实现方法¶

    查找¶

    采用递归的思路，时间复杂度为 \(T(N) = O(d)\) ，其中 \(d\) 为值 \(x\) 的深度：

    BST* FindValue(Type x, BST* t){
    if (t == NULL)
        return NULL;    // not found

    if (x < t->value)
        return FindValue(x, t->left);
    else if (x > t->value)
        return FindValue(x, t->right);
    else:
        return t;   // found
}

    插入¶

    采用递归的思路，但是要注意检查需要插入的值是否已经在数中。

    删除¶

    对于一个要删除的节点，用它的左子树中值最大的节点或者右子树中值最小的节点替代。
