    Lists, Stacks and Queues¶

    0 Abstract Data Type¶

    定义¶

    数据类型 Data Type 指对象和操作的集合。

    抽象数据类型 ADT 是一种经过组织的数据类型，它把对对象和操作的说明与具体表示和实现分离了。

    1 The List ADT¶

    实现一个长度为 \(N\) 的线性表，基本操作包括查找、遍历、插入、删除等。

    1.1 数组 Simple Array Implementation of Lists¶

    • 优点：方便查找。

    • 缺点：需要确定最大长度，不方便插入和删除。

    1.2 链表 Linked Lists¶

    • 优点：方便插入和删除

    技巧

    • 在链表头上加一个空节点 dummy head ，这样所有节点的操作就一致了。

    • 一开始申请好足够的一块内存，速度可以提高，因为每次 malloc 都很费时

    单链表¶

    每个节点包含数据和后继指针。

    双向循环链表¶

    每个节点包含数据和前驱、后继指针，同时头节点的前驱指针指向尾节点，尾节点的后继指针指向头节点。

    十字链表¶

    适合用于存储稀疏矩阵 Sparse Matrix 。

    2 The Stack ADT¶

    2.1 概念¶

    栈是一个后进先出 Last-In-First-Out 的线性表，只允许在栈顶进行插入和删除操作。

    注意

    • 在空栈上执行 pop 操作属于 ADT 错误或逻辑违规。

    • 在满栈上执行 push 操作属于实现错误 Implementation Error，而不是 ADT 错误。

    2.2 实现¶

    链表¶

    一般把链表头部作为栈顶，实现 \(O(1)\) 时间复杂度的插入和删除操作。

    技巧

    可以维护另一个栈作为回收桶，用于存放释放的节点，减少调用 malloc 和 free 带来的系统开销，优化性能。

    数组¶

    通过一块连续的内存空间来存储元素，并使用下标来记录栈顶的位置。

    2.3 应用¶

    符号平衡检查¶

    检查括号是否成对出现且嵌套正确，时间复杂度为 \(O(N)\) 。

    逆波兰表达式¶

    表达式一般有：

    • 前缀 Prefix ，从右向左，遇到数字入栈，遇到操作符弹出两个数字计算后入栈。

    • 中缀 Infix ，需要考虑优先级。

    • 后缀 Postfix （逆波兰表达式），从左往右，遇到数字入栈，遇到操作符弹出两个数字计算后入栈。

    逆波兰表达式无需预知操作符的优先级，计算的时间复杂度为 \(O(N)\) 。

    中缀转后缀的规则是：

    • 遇到数字输出；

    • 遇到操作符，若优先级大于当前栈顶元素，则入栈；反之，则输出当前栈中的元素直到优先级大于栈顶元素，再入栈；

    • 有括号时，遇到左括号立即入栈，遇到右括号则将当前栈中元素弹出并输出直到遇到左括号，注意括号本身弹出后不会输出；

    • 所有数字都输出后，弹出并输出栈中剩下的元素。

    同级别运算符要注意结合性

    例如左结合先弹出 \(a - b - c\) = a b - c - ，而右结合先入栈 \(2^{2^3}\) = 2 2 3 ^ ^ 。

    函数调用¶

    在调用函数时，计算机会在内存的栈区开辟空间，即系统栈。

    3 The Queue ADT¶

    3.1 概念¶

    队列是一个先进先出 First-In-First-Out 的线性表，所有的插入操作都在列表的一端进行，而所有的删除操作都在列表的另一端进行。

    3.2 用数组实现¶

    引入循环队列，可以避免固定数组的空间浪费。
